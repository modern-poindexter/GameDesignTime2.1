# GameDesignTime-2.1

Assignment Overview

For this assignment, you will apply the principles of Rapid Skill Acquisition (RSA) and Lateral Thinking to the process of Game Design. The goal is not to create a perfect or fully polished game. 

You are tasked with designing a simple game that demonstrates a core idea, mechanic, or player experience. As you work, you must also consider how lateral thinking can help you approach the design in unconventional ways. 🧩

IMPORTANT

    If you are a Game Design or Game Programming major, feel free to use a Game Engine for this assignment. Even if you aren't you can do so as well. But no pressure to use one! We've gone over a couple in class so all power in your hands. There must be some type of prototype though!

Deliverables

To successfully meet the requirements, you should have the following:

1. Game Concept

    Short description of the game idea
        Theme/Premise
        End goal of the players 
        3 - 5 sentences

2. Game Mechanic & Rules

    Objective: How players win
    Players: Number of players
    Turn Structure: What players do on their turns
    Core Mechanic: The main action or system driving the game
    3 - 5 bullet sections

3. Lateral Thinking Element

    Explanation of how you used Lateral Thinking in your design
    Examples:
        Introducing randomness
        Using an unexpected mechanic
        Challenging a common assumption

5. Prototype or Visual

    You should form some type of prototype such as the following:
        Board sketch
        Card examples
        Flow diagram of gameplay
        Digital mockup (Canva, Powerpoint, & Figma are great sketching/prototyping tools)
